1. Общие требования к форме регистрации.

Название формы - Registration

Форма предназначена для регистрации на сайте онлайн-магазина оптового заказа пэт бутылок.

1.1 Веб-форма должна быть реализована для работы в браузерах Chrome, Safari, Mozila на следующих операционных системах:
• Microsoft Windows 10 и выше;
• Android 11.x и выше;
• IOS 15.xx и выше;
• MacOS 10.хх и выше.

1.2 Интерфейс регистрации должен поддерживать только английский язык.

2. Требования к дизайну веб-формы

2.1 Дизайн интерфейса регистрации должен адаптироваться под основные разрешения экранов настольных (десктоп), портативных (ноутбук) и планшетных устройств. 

2.2 Элементы интерфейса (поля ввода в формах, кнопки) должны адаптироваться по размеру под устройство, на котором просматривается система.

2.3 Элементы дизайна должны иметь упрощенный вид, но основной функционал должен оставаться доступным.

2.4 Валидация должна происходить при нажатии на кнопку "Submit". Если валидация прошла успешно, то в поле голубого цвета выводится сообщение: "Success! Hello, User!". Если валидация не прошла, то выводится поле красного цвета с сообщением о причине сбоя валидации.

2.5 Поддержка темной темы.

3.  Требования к элементам интерфейса

3.1 Верстка страницы интерфейса регистрации должна быть выполнена в респонсивно-адаптивном виде.

3.2 Форма регистрации должна содержать пять полей: name, surname, email, password, repeat password и кнопку "Submit".

3.3 Обязательными полями заполнения являются: name, email, password, repeat password. 

3.4 Обязательные поля должны быть отмечены звездочкой (*);

3.5 При введении большого количества символов поле не меняет размер;

3.6 При размещении курсора в поле заполнения оно должно подсвечиваться синим цветом по контуру.

3.7 К полю Name должны выполняться следующие требования:
• плейсхолдер поля - Name
• минимально допустимое число знаков 1;
• максимально допустимое число знаков 64;
• допустимые символы - латиница, цифры, спецсимволы (не более одного подряд): пробел ( ), точка (.), тире (-), нижнее подчеркивание (_);
• поле не является регистрозависимым;
• если валидация не прошла, то в выводится поле красного цвета с сообщением "Field "Name" is filled incorrectly";
• после неуспешной валидации введенные данные удаляются.

3.8 К полю Surname должны выполняться следующие требования:
• плейсхолдер поля - Surname
• минимально допустимое число знаков 1;
• максимально допустимое число знаков 64;
• допустимые символы - латиница, цифры, спецсимволы (не более одного подряд): пробел ( ), точка (.), тире (-), нижнее подчеркивание (_);
• поле не является регистрозависимым;
• после неуспешной валидации введенные данные удаляются.

3.9 К полю Email должны выполняться следующие требования:
• плейсхолдер поля - Your email address
• email должен быть уникальным;
• если email не уникальный, выводится поле красного цвета с сообщением "This user is already registered in the system";
• после неуспешной валидации введенные данные удаляются.

идентификатор пользователя

3.10 К полю Password должны выполняться следующие требования:
• плейсхолдер поля - Enter password
• минимально допустимое число знаков 6;
• максимально допустимое число знаков 16;
• допустимые символы - латиница, цифры, спецсимволы (не более одного подряд);
• пароль должен содержать хотя бы одну заглавную буквe, одно число, один спецсимвол;
• копирование данных из поля невозможно;
• если валидация не прошла, выводится поле красного цвета с сообщением "Password is incorrect";
• после неуспешной валидации введенные данные удаляются.

3.11 К полю Repeat password должны выполняться следующие требования:
• плейсхолдер поля - Repeat oassword
• минимально допустимое число знаков 6;
• максимально допустимое число знаков 16;
• допустимые символы - латиница, цифры, спецсимволы (не более одного подряд);
• пароль должен содержать хотя бы одну заглавную буквe, одно число, один спецсимвол;
• копирование данных из поля невозможно;
• если валидация не прошла, выводится поле красного цвета с сообщением "Password mismatch";
• после неуспешной валидации введенные данные удаляются.

3.11 К кнопке "Submit" должны выполняться следующие требования:
• цвет кнопки - зеленый, код #5cb85c;
• при наведении курсора на кнопку цвет кнопки изменяется на #3e8f3e;
• при гажатии курсором на кнопку цвет кнопки меняется на #419641;
• при наведении курсора на кнопку курсор "aero_arrow" изменяется на курсор "aero_link";
• кнопка должна быть активной всегда.
